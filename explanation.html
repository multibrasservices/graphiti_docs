Le Potentiel de votre API Graphiti

Imaginez votre API comme la télécommande d'un cerveau de données intelligent (un "graphe de connaissances"). Ce cerveau ne stocke pas l'information dans des tableaux, mais comme une carte mentale géante où tout est interconnecté. Voici un exemple visuel :

Les Briques de Base de Votre Graphe

Entité (ou Nœud)
C'est un "objet" ou un "concept". Ex: une personne (`Elon Musk`), un film (`Inception`), une entreprise (`SpaceX`).
Relation (ou Arc)
C'est le "lien" qui connecte deux entités et décrit leur rapport. Ex: `Elon Musk` --[est le PDG de]--> `SpaceX`.
UUID (Identifiant Unique)
La "carte d'identité" infalsifiable de chaque entité et relation.
Propriétés
Les "détails" qui enrichissent une entité ou une relation. Ex: `année: 2010`.
Clé d'API
Votre "mot de passe" personnel pour garantir que seul vous pouvez accéder et modifier le graphe.

Répertoire des Endpoints (CRUD)

Voici tous les endpoints de l'API, regroupés par type d'opération.

Create (Créer)
  • POST /messages
    Ajoute une liste de messages.
  • POST /entity-node
    Ajoute une nouvelle entité (nœud).
Read (Lire)
  • POST /search
    Recherche complexe.
  • POST /get-memory
    Récupère la mémoire.
  • GET /entity-edge/{uuid}
    Récupère une entité/relation.
  • GET /episodes/{group_id}
    Récupère les épisodes d'un groupe.
Update (Mettre à jour)
Aucun endpoint de mise à jour n'est défini.
Delete (Supprimer)
  • DELETE /entity-edge/{uuid}
    Supprime une entité/relation.
  • DELETE /group/{group_id}
    Supprime un groupe.
  • DELETE /episode/{uuid}
    Supprime un épisode.
  • POST /clear
    Efface toute la mémoire.
